# 職務経歴書

## ■ プロフェッショナル・サマリー
現場の「技術的な違和感」を放置せず、OSの低レイヤーからアプリケーションのロジックまで、原因を特定するまで掘り下げることを信条とするエンジニア。
特定の言語やフレームワークに依存せず、「運用が回らなくなった複雑なシステム」をシンプルに再構築し、少人数で維持可能な状態へ戻す実績を多く持つ。銀行基盤でのOSバグ特定や、セキュリティ製品間の競合解析など、「仕様書の外側」で起きている事象をエビデンスベースで解決することを得意とし、技術をビジネスの利益（工数削減・安定稼働）へ直結させる。

---

## ■ 技術的専門領域（エンジニアリング・バリュー）
* **深層解析能力:** WinDbgによるメモリダンプ解析、Wiresharkでのパケット解析、OSカーネル挙動の特定。
* **アーキテクチャ再構築:** 複雑化したビジネスロジックの棚卸しと、Python/Go/NoSQLを用いた「変更に強い」システムへの刷新。
* **AI共生型開発:** Claude Code等のAIエージェントを補助輪とした、既存コードの解析高速化および自作ツールの迅速なプロトタイピング。
* **不確実性への対応:** 冪等性（再実行可能性）を担保した設計と、段階的なデプロイメントスキームの構築。

## ■ 主要プロジェクト・ハイライト

### 1. Windows Server 共通基盤におけるDNS切替不具合の根本解決
* **状況:** 「WindowsサーバはDNSセカンダリに自動で切り替わらない」という諦めに近いナレッジが共有され、運用回避が常態化していた。
* **行動:** 「技術的に整合性が取れない」という直感を起点に調査を開始。表面的な設定値は正常だったため、**バイナリレベルでのメモリ/レジストリ解析**を実施。結果、特定バージョンにおけるOS内部データの不整合（OS側のバグ）を特定。
* **結果:** 解析エビデンスを提示し、**全行規模での「開発標準設定」の抜本的な修正**を実現。長年の運用回避を解消し、現場のトラブル対応工数を大幅に削減。

### 2. メモリダンプ解析によるセキュリティ製品の競合回避
* **状況:** EDR製品とマイクロセグメンテーション（Illumio）のDLL競合により、サービスがハングアップする原因不明の事象が発生。
* **行動:** **WinDbgを用いたメモリダンプ解析**を実施し、DLL Injection時の競合箇所を特定。ベンダーが把握していなかった不具合の確証を得て、さらに、非エンジニアでも「問題の構造（自動再起動による無限ループ）」を一瞬で理解できる図解（パワポ1枚）を作成し、ステークホルダー間の認識を統一。技術的根拠に基づいた修正依頼・調整を主導。
* **結果:** EDRのバージョン管理およびVENのIdleモード運用という多層的なワークアラウンドを設計。全社展開を完遂し、ダウンタイムによる損失を回避。

### 3. 標的型メール訓練プラットフォームの抜本的刷新
* **状況:** 複雑化した判定ロジックと、C#/SQL Serverによる重厚な既存システムにより、仕様変更やメンテナンスが技術的負債化していた。
* **行動:** ルールの棚卸しを行いロジックをシンプルに再定義。製品側のCSV仕様変更に柔軟に対応するため、**Python/MongoDBによる弾力的なデータ構造**へ移行。
* **結果:** **運用工数を50%以上削減、担当人数を3名から1名へ適正化。** 新規要件への即時対応が可能な体制を構築。

## ■ 設計・開発における信条（Engineering Philosophy）
* **不確実性を前提とした設計（Design for Failure）**
    「本番環境は検証環境とは異なる挙動をする」「人間は3回に1回はミスをする」という前提に立ち、すべての処理は中断箇所からの再開、あるいは何度実行しても安全な状態を維持する（冪等性）設計を徹底しています。
* **エビデンスに基づく意思決定**
    「おそらくこうだろう」という推測を排し、バイナリ解析やパケットキャプチャといった物理的な証拠からのみ、問題の真因を特定します。
* **「手のかからない」システムの追求**
    技術的なエゴを優先せず、ビジネスロジックを整理し、最小限の人員で安定稼働し続ける「低メンテナンス性」を最優先にアーキテクチャを選定します。
