### \#\# 🛠 Technical Philosophy

1.  **Evidence-Based Debugging**:
    「仕様書の外側」にある真実をバイナリ、メモリダンプ、パケットから特定します。
2.  **AI-Driven Architecture**:
    AIを単なる補完ツールではなく、高度な要件定義を正確な実装へ落とし込むための「実行エンジン」として指揮します。
3.  **Design for Low Maintenance**:
    複雑化したシステムをシンプルに再定義し、運用工数を最小化（3名→1名等）することを最終的なゴールとします。
